帧控制字段应如下指定。 帧类型子字段应设置为指示轮廓宽度命令（0b00）。
制造商特定的子字段应设置为0以发现ZigBee集群中的标准属性，
或者1以在标准或制造商特定集群中发现制造商特定属性。

cluster 0x0000
用于确定设备基本信息的属性和命令，设置用户设备信息（如位置），启用设备并将其重置为出厂默认设置。

这里一个物理ZigBee节点支持多个端点，通常情况下，这些设置中的许多设置将应用于整个节点，也就是说，
它们对于设备上的每个端点都是相同的。 在这种情况下，它们可以为节点实施一次，并映射到每个端点。

为了使cluster 的警报功能可操作，警报cluster 服务器应在相同的端点上实施。

ZCLVersion Attribute
ZCLVersion属性长度为8位，指定该端点上所有群集符合的ZigBee群集库的版本号。
对于这个版本的ZCL，这个属性应该被设置为0x01。

注意：强烈建议将新功能以新群集的形式添加到ZCL，或者向现有群集添加可选属性和可选命令。 
新功能应以“正交”方式添加，利用现有群集执行它们提供的功能，
而不是将这些功能作为新群集的一部分重新实现，
以便通过新群集扩展其功能的设备可以 仍与使用现有群集的设备互通。
在增加ZCL版本时，不应对单个群集的功能进行更改，以防止与同一群集的以前版本进行交互操作。

ApplicationVersion
ApplicationVersion属性的长度是8位，并指定设备中包含的应用程序软件的版本号。 该属性的使用依赖于制造商。

StackVersion属性的长度为8位，并指定设备中包含的ZigBee堆栈实现的版本号。 该属性的使用依赖于制造商。

HWVersion属性的长度是8位，用于指定设备硬件的版本号。 该属性的使用依赖于制造商。

ManufacturerName属性的长度最多为32个字节，并将制造商的名称指定为ZigBee字符串。

ModelIdentifier属性的长度最多为32个字节，并指定制造商指定为ZigBee字符串的型号（或其他标识符）。

DateCode属性是一个最大长度为16字节的ZigBee字符串。 前8个字符根据ISO 8601以国际日期符号，
即YYYYMMDD，例如20060814，指定设备制造商的日期。

PowerSource属性的长度为8位，指定设备可用的电源。 
该属性的位b 0 -b 6表示设备的主电源，位b 7表示设备是否具有电池备份形式的辅助电源。

LocationDescription属性的长度最多为16个字节，并将设备的物理位置描述为ZigBee字符串。
调试过程中可能会将此位置描述添加到设备中。

PhysicalEnvironment属性的长度为8位，并指定设备将在其中运行的物理环境的类型。 
该属性应设置为表3-9中列出的非保留值之一。 除注明外，所有值均适用于支持所有配置文件的端点。



实现此服务器群集的任何端点也应实现基本服务器群集。 要使此群集中所述的警报功能可用，
任何实现Power Configuration服务器群集的端点都必须实现警报服务器群集（请参阅子句子警报（群集））。

为方便起见，本规范中定义的属性被排列成相关属性集; 
每个集合最多可以包含16个属性。 属性标识符被编码，
使得最重要的三个节点指定属性集，最低有效位半字节指定集合内的属性。 
表3-14列出了当前定义的属性集。

场景标识符0x00与组标识符0x0000一起保留给OnOff集群使用的全局场景。

在通过组播或广播传输服务接收到添加场景，查看场景，移除场景，
移除所有场景，存储场景，增强场景，增强场景或复制场景命令帧时，不得回应。

每个扩展字段集的格式是一个携带集群ID的16位字段，
后跟一个8位长度字段和在相关集群中指定的一组场景扩展字段。 
长度字段保存该扩展字段集的八位字节长度。

包含在为每个群集设置的扩展字段中的属性在本文档中的该群集的规范中定义（ZigBee群集库）。 
字段集由按照集群规范中给定的顺序连接在一起的这些属性的值组成，不包含属性标识符或数据类型指示符。

对于前向兼容性，接收该命令应允许未来可能将其他属性添加到集群规范中给出的列表尾部（通过忽略它们）。
同样，它应允许从这些清单的尾端删除一个或多个属性（见3.7.2.4.7.2）。

Store Scene
接收到该命令后，设备应该（如果可能的话）向场景表添加一个条目，
其中包含命令中给出的场景ID和组ID，以及所有与设备上其他簇的当前状态相对应的扩展域集。

如果一个条目已经具有相同的场景ID和组ID，由于之前的添加场景命令，扩展字段集被覆盖（即完全替换），
但过渡时间字段和场景名称字段是 保持不变。
如果不存在这样的条目，则过渡时间字段应被设置为0，并且场景名称字段应被设置为空字符串。
